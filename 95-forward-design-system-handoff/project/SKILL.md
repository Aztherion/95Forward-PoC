If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand.

**The soul of this brand is transparency + warmth + calm focus.** Hold these in mind whatever you build:
- The copilot **proposes; the human decides.** AI is the reserved iris voice, always provisional (approve / edit / dismiss), always with provenance. Never silently applied, never sci-fi (no glow/sparkles/gradients-as-intelligence).
- **"Unknown" is honorable** — a research gap, written as an invitation, never a red error or empty field.
- A coach's copy voice: plain, encouraging, dignified; active verbs ("Plan the visit", "Log the ask"); sentence case; **no emoji**.
- Two registers from one system: **A** (portfolio/analysis — calm dense cockpit) and **B** (in the moment — large, quiet, low chrome). Shared tokens, two densities.
- Spend boldness on the **QPIScore** (the number you can see inside); keep everything around it quiet.

If the user invokes this skill without any other guidance, ask them what they want to build or design, ask some questions, and act as an expert designer who outputs HTML artifacts _or_ production code, depending on the need.
